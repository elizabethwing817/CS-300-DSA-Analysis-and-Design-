# CS-300-DSA-Analysis-and-Design-

Problem Overview

The primary problem addressed in this course was designing and implementing a program that efficiently stores, sorts, and retrieves course information for academic advisors in the Computer Science department at ABCU. The software needed to read course data from a file, organize it using appropriate data structures, print courses in alphanumeric order, and display prerequisite information for selected courses. These projects emphasized both algorithmic efficiency and practical usability.

Approach and Use of Data Structures

I approached this problem by first analyzing different data structures and their performance trade-offs, including vectors, hash tables, and binary search trees. Understanding data structures was essential because each structure impacts run-time efficiency, memory usage, and scalability. After evaluating these factors, I selected the structure that best supported sorted output and efficient searching. I focused on writing algorithms that balanced performance with clarity, ensuring that the program met functional requirements while remaining easy to understand and maintain.

Overcoming Challenges

One of the main challenges I encountered was implementing file parsing and correctly inserting data into the chosen data structure while maintaining order. Debugging pointer logic and ensuring accurate traversal of the structure required careful testing and iteration. I overcame these challenges by breaking the problem into smaller steps, validating data at each stage, and using runtime analysis to confirm that the program behaved as expected. Reviewing errors methodically and refactoring code incrementally helped me resolve issues without introducing new bugs.

Impact on Software Design Thinking

This project significantly expanded how I approach software design. I now place greater emphasis on planning data structures before writing code and considering performance early in the design process. Rather than focusing solely on making a program work, I think more critically about how well it scales, how efficiently it runs, and how easily it can be modified in the future. This structured approach has made my development process more intentional and effective.

Writing Maintainable, Readable, and Adaptable Code

Through this course, I have improved my ability to write code that is maintainable and adaptable. I now use clearer variable names, modular functions, and consistent formatting to improve readability. I also design programs with flexibility in mind, making it easier to update logic or replace data structures if requirements change. These practices ensure that my code is not only functional today but also usable and understandable in the future.
